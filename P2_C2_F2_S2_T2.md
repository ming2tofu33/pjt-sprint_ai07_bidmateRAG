## 3. 심층 분석 및 인사이트

### 지표 하락의 원인 분석
1. **단답형 구조의 한계**:
   - P2 프롬프트는 `결론 중심 단답`을 요구함. LLM이 답변을 간결하게 축약하면서, 판정 모델(Judge)은 이를 "충분한 정보를 제공하지 못함"으로 인식하여 `Relevance` 점수를 낮게 부여함.
2. **페르소나(감사관)의 영향**:
   - "깐깐한 수석 감사관" 페르소나가 적용되어 답변이 매우 보수적으로 작성됨. 조금이라도 불확실한 경우 답변을 생략하거나 극도로 줄이는 경향이 있어 정보의 풍부함이 감소함.
3. **페이지 번호 제약**:
   - 프롬프트에서 `페이지 번호 명시`를 강제했으나, 실제 검색된 텍스트 청크에 페이지 번호 정보가 누락되거나 불분명할 경우 모델이 근거를 제시하는 데 어려움을 겪음.

### 효율성 측면
- 답변의 길이가 짧아짐에 따라 **평균 응답 시간(Latency)이 P1 대비 약 9초 가량 단축**됨. 속도가 중요한 서비스라면 고려할 만한 옵션이나, 품질 지표와의 트레이드오프(Trade-off)가 발생함.

## 4. 향후 실험 방향
- **P3 실험**: P2의 단답형보다는 풍부하면서 P1보다는 정제된 형태의 프롬프트를 시도하여 품질과 속도의 균형점을 찾을 예정.
- **판정 모델의 '단답' 이해도**: 단답형 답변에 대해 판정 모델이 더 공정하게 평가할 수 있도록 평가 기준(Prompt) 조정 검토.
