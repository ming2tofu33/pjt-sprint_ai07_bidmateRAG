### 슬라이드 6 — "MRR 0.000 → 1.000" Judge v2 드라마 & 구조 ★

> **Hook**: "측정 도구의 신뢰성이 곧 성능 지표의 신뢰성"

**핵심 텍스트**:
- Judge v1 시절: LLM이 Pass/Fail 주관 응답 → 4회 연속 MRR=0.000 기록 (채점 파싱 실패)
- Judge v2 전환 후 즉시 MRR 1.000 정상화 — 파이프라인이 아니라 측정 도구가 문제였음
- v2 구조: LLM은 근거(Evidence)·추론(Reasoning)만 추출 / 점수는 코드가 5단계 비율제(0/0.25/0.5/0.75/1.0) 계산

**수치/표**:
| 항목 | v1 | v2 |
|---|---|---|
| 채점 방식 | LLM 주관 점수 | 코드 비율제 계산 |
| 지표 | Pass/Fail | 5대 ratio score |
| 결과 | 4회 연속 MRR 0.000 | 즉시 MRR 1.000 |
| 투명성 | 불가 | 채점 과정 로깅 |

**시각화 제안**: 시계열 꺾은선 그래프 (날짜 × MRR) + v1/v2 구조 대비 다이어그램

**각주**: 커밋 4867df2 `feat: add LLM judge v2 with evidence-based ratio scores`

### 슬라이드 17 — 반직관 발견: 단락 기반이 Semantic Chunking을 이겼다 ★★

**시각화 제안**: 4종 청킹 전략 가로 막대 그래프 — MRR은 모두 같은 길이로 평행, Faithfulness만 차이 강조
