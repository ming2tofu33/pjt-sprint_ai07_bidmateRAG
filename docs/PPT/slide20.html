Cross-Encoder를 본 경로에서 분리한 3가지 이유

Dense → Dense+Sparse+RRF Hybrid 전환 후, Cross-Encoder 추가 여부를 검토 — 3가지 이유로 본 경로에서 분리

이유 ①

운영 환경 제약

시나리오 A의 L4 24GB VRAM에서 로컬 LLM + CE 동시 로드 불가

VRAM 할당 ·

Gemma-4-E4B (INT4) ≈ 10GB
+ CE model ≈ 2GB + KV cache → 24GB 초과

이유 ②

기여도 미미

Hybrid(Dense+BM25+RRF) 이후 CE 추가 재정렬의 개선 폭이 작음

역설 ·

CE OFF로 오히려 Faithfulness +4.4% / Recall +5.6% — 본 경로에서 CE의 한계 효용이 음수

이유 ③

공정한 A/B 비교

검색단을 통제 않고 B에만 CE를 얹으면 LLM 효과 + CE 효과가 뒤섞임

설계 원칙 ·

검색단은 A/B 공통 → LLM(gpt-5-mini vs Gemma-4-E4B)만 교체해 순수 비교
