멀티턴 재작성 안전장치

후속 질문 재작성은 편의성을 높이지만, 수치 정보가 포함되면 검색 범위가 잘못 좁혀질 수 있다

1차 적용

안전 슬롯 (rewrite 포함)

대명사 해석에 필요한 식별 정보만 전달

발주기관 "국민연금공단"
사업명 "이러닝 시스템 구축"
관심 속성 "보안 요구사항"
위험 슬롯 (rewrite 제외)

숫자·금액·일정은 프롬프트에 포함하면 검색 오염

예산 "352,000,000원"
일정·연도 "2024년 7월 10일"
평가기준 배점 "기술평가 80점"

Rewrite 검증·폴백 플로우

① 입력

안전 슬롯만
LLM에 전달

② Rewrite

LLM이 대명사
치환한 쿼리 생성

③ 검증 (핵심!)

원문·최근 턴에 없던
숫자·금액 주입 탐지

④ 폴백

validation_failed
→ rule fallback
